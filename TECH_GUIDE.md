# 🎯 ЧТО ТАКОЕ ЭТОТ БОТ?

**Простыми словами:** Это автоматический продавец в Telegram, который работает 24 часа в сутки, 7 дней в неделю, без перерывов и выходных.

## Что умеет бот:

✅ **Показывать каталог товаров** — 11 категорий, карточки с фото и ценами

✅ **Принимать заказы** — пользователь выбирает товар, оплачивает, получает подтверждение

✅ **Вести корзину** — можно добавить несколько товаров, изменить количество

✅ **Подбор товара** — фильтры по размеру, цвету, цене, материалу

✅ **Админ-панель** — управление товарами, заказами, текстами, администраторами

✅ **Работать автономно** — бот живёт на сервере Zo Computer и не зависит от вашего ПК

## Для кого этот бот:

👤 **Покупатели** — выбирают и заказывают товары

👨‍💼 **Администраторы** — управляют магазином через бота

🔐 **Суперадмин** — главный администратор, может добавлять других админов

# 🏗 КАК ВСЁ УСТРОЕНО ВНУТРИ

## Три уровня работы бота:

```
┌─────────────────────────────────────┐
│     ПОЛЬЗОВАТЕЛЬ (Telegram)         │
│   Нажимает кнопки, пишет команды    │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     БОТ (Python + aiogram)          │
│   Обрабатывает сообщения, логика    │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     БАЗА ДАННЫХ (SQLite)            │
│   Хранит товары, заказы, настройки   │
└─────────────────────────────────────┘
```

## Технологии (простыми словами):

🐍 **Python** — язык программирования, на котором написан бот

📦 **aiogram** — библиотека для работы с Telegram (как "переводчик" между ботом и Telegram)

💾 **SQLite** — база данных (как Excel, но умнее)

🖥 **Zo Computer** — сервер, где живёт бот (работает 24/7)

# ⚙️ КАК РАБОТАЕТ БОТ ТЕХНИЧЕСКИ

## Цикл работы бота:

```
1️⃣ ПОЛЬЗОВАТЕЛЬ пишет сообщение
         ↓
2️⃣ TELEGRAM отправляет боту
         ↓
3️⃣ БОТ обрабатывает (handlers)
         ↓
4️⃣ БАЗА ДАННЫХ сохраняет/читает
         ↓
5️⃣ БОТ отправляет ответ
         ↓
6️⃣ TELEGRAM показывает пользователю
```

## Обработка сообщений:

**Шаг 1: Пользователь нажал кнопку "Каталог"**

```python
# В handlers_user.py
@router.callback_query(F.data == "catalog")
async def show_catalog(call: CallbackQuery):
    # Бот получает список категорий из БД
    categories = await db.list_categories()
    # Бот показывает пользователю
    await call.message.answer("Каталог", reply_markup=kb.categories_kb(categories))
```

**Шаг 2: Пользователь выбрал категорию "Халаты"**

```python
@router.callback_query(F.data.startswith("cat:"))
async def show_category(call: CallbackQuery):
    category_id = int(call.data.split(":")[1])
    # Бот получает товары этой категории
    products = await db.list_products(category_id)
    # Показывает товары
    await call.message.answer("Товары", reply_markup=kb.products_kb(products))
```

**Шаг 3: Пользователь выбрал товар**

```python
@router.callback_query(F.data.startswith("prod:"))
async def show_product(call: CallbackQuery):
    product_id = int(call.data.split(":")[1])
    # Бот получает информацию о товаре
    product = await db.get_product(product_id)
    # Показывает карточку товара с фото
```

### 3️⃣ **Суперадмин** (ваш ID: `7582523959`)
- Может: ВСЁ, включая управление другими админами

## Как добавить админа:

**Вариант 1: Через бот-панель (только суперадмин)**
1. Откройте бот
2. Напишите `/admin`
3. Нажмите "👥 Администраторы"
4. Нажмите "➕ Добавить админа"
5. Введите ID пользователя

**Вариант 2: Узнать ID пользователя**
1. Пользователь пишет боту `/myid`
2. Бот показывает его ID
3. Админ копирует ID

# 🆘 ЧАСТЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

## ❌ Проблема: Бот не отвечает

**Возможные причины:**

1. **Бот запущен в двух местах одновременно**
   - Симптом: В логах ошибка `Conflict: terminated by other getUpdates request`
   - Решение: Закройте бот на ПК, перезапустите на сервере

2. **Неверный токен**
   - Симптом: Бот вообще не стартует
   - Решение: Проверьте токен в `.env`

3. **Нет интернет-соединения**
   - Симптом: Бот работал, потом перестал
   - Решение: Проверьте интернет на сервере

## ❌ Проблема: Товары не показываются

**Возможные причины:**

1. **База пуста**
   - Решение: Запустите `python seed_demo.py` для тестовых товаров

2. **Нет фото у товара**
   - Решение: Через админку добавьте фото товару

## ❌ Проблема: Админ-панель не открывается

**Возможные причины:**

1. **Вы не админ**
   - Решение: Суперадмин должен добавить ваш ID

2. **Неправильный ID в конфиге**
   - Решение: Проверьте `config.py` → `ADMIN_IDS`

## ❌ Проблема: Заказы не приходят админу

**Возможные причины:**

1. **Админ заблокировал бота**
   - Решение: Разблокируйте бота

2. **Неверный ID админа**
   - Решение: Проверьте через `/myid`
